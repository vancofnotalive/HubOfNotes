Clean Architecture - Update Method

Explanation:
  • In Clean Architecture, it is recommended that in a generic repository you do not declare an update method.
  • Reason: The update method's logic and parameters can vary depending on the specific needs of a controller or service.
  • Solution: Instead of a generic update method, you should define it in a specific repository for the particular use case. This allows more flexibility in how the update method behaves.
  • The generic repsitory should contains methods only using or having need of < T >
  • Mostly it is recommended that you put save() method in specific repository(specifc interface) as well
