Rules of Clean Architecture

Four Projects for an Application

  • ProjectName.Domain
  • ProjectName.Infrastructure
  • ProjectName.Application
  • ProjectName.Presentation

A controller should never access or have application db instead it should use custom methods made

Benefits of Clean Architecture

  • Less dependent on others
  • Easy maintainability and scalability

If the client asks to not use entity framework, then this will be easy to make changes in the application layer/project and there will be no need to change other layers/projects.
