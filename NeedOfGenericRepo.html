Why Use a Generic Repository in Clean Architecture?

The need for a generic repository arises to avoid creating separate interfaces and classes for implementing common methods in each controller. Instead, you can create a repository containing the following common methods:

  • CRUD operations (Create, Read, Update, Delete).

Specific Methods

For methods specific to a controller (that cannot be implemented in the generic repository), you create:

  • A specific interface.
  • A class implementing the interface for that controller.

Dependency Injection Registration

To use a generic repository, you must register it in dependency injection using one of the following methods:

  • AddScoped
  • AddSingleton
  • AddTransient
